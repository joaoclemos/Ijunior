# 🚀 Trainee iJunior - Jornada de Desenvolvimento

Seja bem-vindo(a) ao repositório central das minhas entregas durante o processo de Trainee da **iJunior (UFMG)**. Este espaço documenta minha evolução técnica, desafios superados e a consolidação de conceitos fundamentais no desenvolvimento de software.

---

## 👨‍💻 Sobre Mim
Estou em fase de capacitação na iJunior, focado em aprender as melhores práticas do mercado, desde a organização de fluxo de trabalho com Git até o desenvolvimento de sistemas escaláveis com TypeScript.

---

## 🛠️ Competências Adquiridas

### 🧬 Desenvolvimento Web & Lógica
* **Manipulação do DOM:** Criação de interfaces dinâmicas e vivas que reagem às ações do usuário sem recarregar a página.
* **POO (Programação Orientada a Objetos):** Organização de código através de classes, construtores e métodos, garantindo modularidade e reutilização.
* **TypeScript:** Desenvolvimento com tipagem forte, garantindo maior segurança e menos bugs durante o desenvolvimento.

### ⚙️ Ferramentas & Processos
* **Versionamento com Git/GitHub:** Domínio de fluxos profissionais (Gitflow), utilização de branches (`main`, `desenvolvimento`, `feature`), e commits padronizados.
* **Arquitetura de Software:** Separação clara entre estrutura (HTML), estilo (CSS) e lógica (TS).

---

## 📅 Histórico de Entregas

### [Semana 2] - Fundamentos de TypeScript (Lógica & Tipagem)
* **O que foi feito:** Resolução de uma bateria de exercícios práticos focados em lógica de programação.
* **Aprendizados:** Tipos primitivos (`string`, `number`, `boolean`), estruturas condicionais, loops (`for`/`while`), interfaces para modelagem de objetos e funções com callbacks.
* **Destaque:** Transição da mentalidade de JavaScript dinâmico para o desenvolvimento tipado e seguro com TS.

### [Semana 3] - Página de Perfil & Gitflow Profissional
* **O que foi feito:** Criação de uma página de portfólio pessoal utilizando HTML5 semântico e CSS3 (Flexbox/Grid).
* **Aprendizados:** Implementação do **Gitflow completo** da iJunior. 
* **Fluxo de Trabalho:** - Gerenciamento de múltiplas branches: `main`, `homologacao` e `desenvolvimento`.
    - Trabalho em `feature branches` isoladas.
    - Aplicação rigorosa de **Conventional Commits** (`feat:`, `style:`, `fix:`).
    - Abertura e documentação de Pull Requests.
* **Tags:** `#HTML5` `#CSS3` `#Gitflow` `#CleanCode`

### [Semana 4] - iTask Manager (Manipulação de DOM & POO)
* **O que foi feito:** Um gerenciador de tarefas dinâmico que cria instâncias de uma classe `Tarefa`.
* **Principais Desafios:** Integrar a lógica de classes com a criação de elementos HTML em tempo real e garantir o feedback visual imediato ao usuário (check-task).
* **Tags:** `#TypeScript` `#POO` `#DOM` `#CSS3`

## 🚀 Como Executar os Projetos

Como este repositório utiliza **TypeScript**, para rodar as aplicações locais é necessário:

1. Clonar o repositório: `git clone https://github.com/SEU-USUARIO/NOME-DO-REPO.git`
2. Instalar o TypeScript globalmente (caso não tenha): `npm install -g typescript`
3. Na pasta do projeto desejado, compilar os arquivos: `tsc -w`
4. Abrir o arquivo `index.html` no seu navegador preferido.
