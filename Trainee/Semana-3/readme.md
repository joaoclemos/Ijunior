## 🚀 A Entrega da Semana

O objetivo é criar uma página web estática e entregá-la usando o **fluxo GitFlow completo da iJunior**, simulando um ambiente profissional (mesmo trabalhando sozinho).

### 📝 Descrição do desafio

- Você deve criar uma **"Página de Perfil"** ou **"Portfólio Pessoal"** simples.
Esta página deve ser um único arquivo `index.html`, um `style.css`  e caso tiver imagens, uma pasta `images` contendo-as. Ela deve conter seções como "Sobre Mim", "Meus Projetos" (podem ser fictícios/ da faculdade/ pessoais), e "Contato".

### ✅ Critérios de aceite

Para que sua entrega seja aprovada, ela deve:

- **[HTML/CSS]** A página deve ser estruturada com HTML semântico e usar Flexbox/Grid.
- **[Git]** O projeto deve estar em um **novo repositório** no seu GitHub.
- **[Git]** O repositório deve conter as três branches principais: `main`, `desenvolvimento` e `homologacao`.
- **[Git]** A branch `desenvolvimento` deve ser definida como a **branch padrão** do repositório no GitHub.
- **[Git]** O desenvolvimento deve ser feito em uma `feature` branch (ex: `feature/pagina-perfil`) criada a partir da `desenvolvimento`.
- **[Git]** Os commits devem seguir o padrão **Conventional Commits**.
- **[Git]** Um **Pull Request (PR)** deve ser aberto da sua branch `feature` para a branch `desenvolvimento`.
